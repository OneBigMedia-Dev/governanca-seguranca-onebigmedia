Como usar este material

Este pré-curso existe para tirar o "atrito técnico" do caminho. A ideia é que, no primeiro encontro, ninguém perca tempo instalando coisas — todo mundo já entra pronto para pensar em soluções.

Siga de cima pra baixo

Fundamentos: IA, LLMs e Claude Code

Quatro ideias-base para você chegar com o vocabulário certo. Leitura de ~8 minutos. Não precisa memorizar — só entender o espírito da coisa.

01 O que é um LLM

Um "Large Language Model" (como o Claude) é um modelo treinado para prever e gerar texto. Ele não "pesquisa" respostas prontas — ele monta a resposta com base em padrões. Por isso é ótimo em linguagem e raciocínio, mas pode errar com confiança. A revisão humana é sempre parte do processo.

02 O que é um bom prompt

Prompt é a instrução que você dá. Bons prompts são específicos: dizem o contexto, o objetivo, o formato desejado e dão exemplos. "Resuma isso" rende menos que "Resuma este relatório em 5 tópicos, foco em custos, para enviar à diretoria". Clareza vale mais que esperteza.

03 O que é o Claude Code

É o Claude com "mãos": além de conversar, ele lê seus arquivos, escreve e roda código, cria automações e mexe no seu projeto — sempre pedindo sua confirmação em ações importantes. É a ponte entre "ter uma ideia" e "ter algo funcionando".

04 Você continua no comando

A IA acelera, mas quem decide é você. Seu papel muda de "fazer tudo à mão" para orientar, revisar e validar. Entender o que a solução faz é o que separa uma automação confiável de uma caixa-preta arriscada.

↻ O ciclo de trabalho com um agente

Na prática, construir com o Claude Code é um ciclo curto que se repete — você guia, ele executa, você revisa:

Você descreve o objetivo
→
Claude propõe um plano
→
Claude executa
→
Você revisa & ajusta
↺

Como vamos operar Claude, Claude Code e VS Code

O curso não é sobre decorar comando. É sobre saber escolher a interface certa, pedir plano antes de execução, revisar o que a IA fez e transformar uma ideia em solução controlada.

APP Claude App e chat

Use para pensar, organizar ideias, resumir contexto, melhorar prompt, simular perguntas, escrever documentação e preparar roteiro de apresentação. É o lugar para raciocinar antes de mexer no projeto.

CODE Claude Code no VS Code

Use quando a tarefa envolve arquivos: criar pasta, ler código, editar README, gerar script, rodar teste, montar app Next.js, revisar diff e documentar decisões.

PLAN Plan Mode

Use quando o Claude ainda não deve editar nada. Primeiro ele explora, entende a pasta, propõe etapas e pede aprovação. Só depois o trabalho vira execução.

PROMPT Prompt engineering prático

Todo pedido deve deixar claro: contexto, objetivo, restrições, formato da entrega, critérios de sucesso e o que não pode ser feito. Prompt bom reduz retrabalho e risco.

DIFF Revisão humana

Antes de aceitar resultado, vamos olhar arquivos alterados, comandos executados, dados usados e próximos passos. A IA acelera, mas a responsabilidade continua humana.

COWORK Modo coworker

Trate o Claude como par de trabalho: peça alternativas, questione decisões, mande ele explicar o que fez, peça checklist e interrompa quando o caminho não fizer sentido.

Governança, segurança e uso de IA

Construir com IA vem com responsabilidade. Este é um resumo prático da Base de Governança de Segurança da OneBigMedia (v1.0) — o que você precisa saber antes de começar. O documento completo é a fonte oficial e vale a leitura.

Quem é quem: a validação de projetos e a aprovação para produção passam pelo Paulo. O acesso ao Claude Code é liberado pelo Fred Valente. Toda demanda relevante nasce e é registrada no Jira.

✓ Princípios que guiam tudo

  • Menor privilégio: só o acesso necessário, nada além.
  • Revisão humana é obrigatória — nada crítico entra em produção sem ela.
  • Segredos nunca ficam no código (tokens, chaves, senhas, .env).
  • O que é crítico precisa ser rastreável: Jira, Pull Request e logs.
  • A IA acelera, mas não substitui a responsabilidade técnica de quem constrói.

◈ Regras de uso de IA

  • Código gerado por IA passa por Pull Request e revisão humana.
  • Mudanças de IA em produção seguem o mesmo fluxo de qualquer mudança.
  • Não conceda acesso administrativo a agentes de IA por padrão.
  • Agentes não executam comandos destrutivos sem aprovação explícita.
  • Conectores, MCPs e plugins que acessam dados internos precisam ser aprovados antes.

🔒 Nunca envie para IA sem aprovação

  • Senhas, API keys, tokens OAuth/refresh e service account keys.
  • Arquivos .env, client_secret.json, chaves .pem/.key.
  • Dumps de banco, dados de clientes, parceiros ou financeiros.
  • Credenciais de Google, GCP, GitHub, Jira, YouTube CMS ou bancos.
  • Logs com tokens, cookies ou dados pessoais.

✓ Pode usar, sem aprovação extra

  • Trechos de código que não contêm segredos.
  • Logs anonimizados e mensagens de erro sem credenciais.
  • Documentação técnica interna sem dados sensíveis.
  • Dados fictícios ou mascarados.

▤ Classifique o risco do seu projeto

Todo projeto, app ou integração é classificado antes de avançar. Na dúvida, trate como o nível mais alto — isso define quantas aprovações ele precisa.

BAIXO

Sem dados sensíveis, sem credenciais, não toca produção. Ex.: script local, prova de conceito, documentação.

MÉDIO

Usa APIs externas, tokens limitados ou dados internos; pode gerar custo. Ex.: automação em teste, pipeline em homologação.

ALTO

Toca produção, banco real, dados de clientes/financeiro ou tem permissões administrativas. Exige mais aprovações.

C Regras do Claude Code

  • Acesso liberado pelo Fred Valente — use sua conta individual.
  • Não compartilhe conta, sessão ou chave de acesso.
  • Use permissões restritas por padrão e revise os comandos antes de aprovar a execução.
  • Sem execução automática irrestrita em repositórios críticos.
  • Em projeto crítico: sessão separada por projeto e registro no Jira.

$ Ferramentas, custos & Jira

  • Não adote um app, SaaS ou extensão novos sem registrar e aprovar no Jira.
  • Todo uso relevante de IA tem dono, finalidade e limite de custo definidos.
  • Repositórios ficam na organização OneBigMedia-Dev, com main protegida e PR obrigatório.
  • Nenhum projeto vai para produção sem passar pelo pipeline de validação.
