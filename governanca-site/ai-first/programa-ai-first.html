Faça na ordem

~/ai-first/ferramentas --kit

Seu kit de ferramentas.

Este e o conjunto minimo para acompanhar o programa: pensar com Claude, operar arquivos no VS Code, criar apps simples, tratar dados e manter governanca sobre acessos, segredos e fontes internas.

Claude App e chat

Usado para raciocinar, comparar caminhos, montar prompts, revisar escopo e trabalhar em modo coworker antes de mexer em arquivos.

Claude Code CLI

Usado dentro da pasta do projeto para ler contexto, planejar, editar arquivos, executar comandos aprovados e explicar diffs.

Claude Code no VS Code

Integra o Claude ao editor para trabalhar com arquivos abertos, terminal, revisao visual e fluxo Plan - Execute - Review.

VS Code

Editor principal do curso. E onde vamos abrir a pasta, organizar arquivos, rodar terminal e revisar tudo que a IA alterar.

Git

Historico do projeto. Ajuda a comparar alteracoes, registrar evolucao e voltar atras quando algo nao ficar correto.

Node.js e Next.js

Base para criar telas e pequenos apps internos. O Next.js entra como forma pratica de transformar automacao em interface.

Python

Linguagem para scripts, planilhas, arquivos, automacoes e analises. O foco e criar rotinas pequenas, legiveis e revisaveis.

Banco de dados e SQL

Usado apenas com acesso aprovado, menor privilegio e, de preferencia, leitura. Para treino, usamos SQLite, base exemplo ou dados mascarados.

Terminal

Porta de entrada para comandos, validacoes, instalacoes e execucoes locais. Tudo que for sensivel precisa ser revisado antes de rodar.

Regra: nenhuma ferramenta, extensao, conector, token, banco ou automacao com custo deve ser usada fora do fluxo de aprovacao da One Big Media.

~/ai-first/fundamentos --operacao

Fundamentos e modo de operacao.

O programa nao exige que todo mundo vire programador. Ele exige clareza para orientar a IA, revisar resultado e saber quando uma automacao pode ou nao seguir adiante.

LLM sem misterio

Claude gera respostas a partir de padroes e contexto. Ele raciocina bem, mas pode errar com confianca. Por isso revisao humana faz parte do fluxo.

Prompt bom

Inclui contexto, objetivo, restricoes, formato desejado, exemplos e criterio de sucesso. Quanto mais especifico o pedido, menor o retrabalho.

Claude Code

E o Claude com acesso ao projeto: le arquivos, propõe plano, edita codigo, roda comandos aprovados e explica o que mudou.

Plan Mode

Use quando ainda nao quer editar nada. Primeiro o agente explora, entende a pasta, propõe etapas e so depois executa.

Review

Antes de aceitar resultado, revise arquivos alterados, comandos executados, dados usados, diffs e proximos passos.

Coworker

Trate Claude como par de trabalho: peça alternativas, mande explicar decisoes, interrompa caminhos ruins e valide saidas com criterio.

Memoria do projeto

Use README, docs e, quando fizer sentido, CLAUDE.md para registrar padroes, comandos, decisoes e limites recorrentes.

Dados com contexto

Explique origem, permissao, mascara, qualidade e limite de uso. Dado real sensivel nao entra em prompt sem aprovacao.

Humano decide

A IA acelera o trabalho, mas a responsabilidade tecnica e de negocio continua com quem constrói e valida o MVP.

~/ai-first/governanca --seguranca --dados

Governanca, seguranca e uso de IA.

Todo MVP precisa nascer com limites claros. A regra e simples: menor privilegio, revisao humana, rastreabilidade e nenhum segredo exposto para IA.

Quem e quem: a validacao de projetos e a aprovacao para producao passam pelo Paulo. O acesso ao Claude Code e liberado pelo Fred Valente. Demandas relevantes devem nascer ou ser registradas no Jira.

Principios obrigatorios

  • Menor privilegio por padrao.
  • Revisao humana antes de mudanca critica.
  • Segredos nunca ficam no codigo, prompt, print ou log.
  • Projeto critico precisa de Jira, PR e registro de decisao.

Nunca envie para IA

  • Senhas, tokens, API keys e service accounts.
  • Arquivos .env, client_secret, .pem ou .key.
  • Dumps de banco e dados de clientes, parceiros ou financeiro.
  • Logs com cookies, credenciais ou dados pessoais.

Pode usar sem aprovacao extra

  • Codigo sem segredo.
  • Logs anonimizados.
  • Dados ficticios, minimizados ou mascarados.
  • Documentacao tecnica sem informacao sensivel.

Claude Code

  • Use conta individual liberada pela empresa.
  • Revise comandos antes de executar.
  • Sem auto-execucao irrestrita em repositorios criticos.
  • Projeto critico pede sessao separada e registro.

Ferramentas e custo

  • Nao adote SaaS, extensao, conector ou MCP sem aprovacao.
  • Todo uso relevante precisa de dono, finalidade e limite de custo.
  • Repositorios oficiais ficam na organizacao aprovada.
  • Producao exige pipeline de validacao.

Banco de dados

  • Preferir treino com SQLite, CSV ou base mascarada.
  • Banco real apenas com aprovacao e menor privilegio.
  • Credencial corporativa nunca entra em prompt.
  • Consulta e resultado precisam ser documentados.
Risco baixo

Sem dados sensiveis, sem credenciais e sem producao. Ex.: script local, prova de conceito, documentacao.

Risco medio

Usa APIs externas, tokens limitados ou dados internos. Ex.: automacao em teste, pipeline em homologacao.

Risco alto

Toca producao, banco real, cliente, financeiro ou permissao administrativa. Exige aprovacao reforcada.

~/ai-first/desafios --por-area

Cada pessoa traz uma dor real da area.

A escolha do desafio e o ponto mais importante do programa. O melhor caso nao e o mais complexo: e aquele que tem impacto claro, dados acessiveis, risco controlado e alguem disposto a validar o resultado.

Como vamos avaliar cada desafio

  • Repeticao: acontece toda semana, todo mes ou consome muitas horas?
  • Impacto: reduz erro, economiza tempo, melhora receita, protege informacao ou acelera decisao?
  • Dados: os insumos existem e podem ser usados com seguranca?
  • Escopo: cabe no ciclo de 8 encontros principais?
  • Risco: envolve producao, dados pessoais, financeiro, clientes, CMS, GCP, GitHub ou credenciais?
  • Dono: existe alguem da area que valide se a solucao realmente funciona?
Area e responsavelQuem traz o desafio e quem valida o resultado.
Processo atualComo e feito hoje, com quais ferramentas e quanto tempo consome.
Dor principalErro, retrabalho, atraso, custo, risco ou oportunidade perdida.
Dados envolvidosPlanilhas, arquivos, APIs, bancos, relatórios, e-mails ou sistemas.
Resultado esperadoO que precisa existir ao final para considerar o MVP util.
Limites de segurancaO que nao pode ser exposto, automatizado ou acessado sem aprovacao.
